1. Observable as Infinite Stream
-------------------------------------------------------------------------------------------------------
Observable is an infinite list of pairs of Time T and value a: [{T, a}..].

An Observable is an infinite stream, formally defined as a potentially unbounded sequence 
of pairs representing values (a) emitted over discrete points in time T:
Observable = [{T,a..}]

RxJS makes Events first-class citizens in the language.:

RxJS promotes Events to first-class citizens, representing them as observable streams. 
This enables composability through powerful functional programming constructs, 
allowing streams of events to be manipulated, passed around, and composed seamlessly.

RxJS Observable is a unifying interface that makes no assumption on the structure of the underlying datasource.

The RxJS Observable interface abstracts the specifics of underlying data sources 
(arrays, Promises, DOM events, web sockets, etc.), 
providing a consistent and composable model regardless of the source type.

RxJS is the intersection of the Iterator and Subject/Observer Pattern.

Refined Statement:

RxJS integrates both the pull-based Iterator Pattern and the push-based Observer Pattern, 

thereby creating a unified approach capable of handling both synchronous and asynchronous data streams.
